# Tender Intelligence Platform

## Mission

Build an enterprise-grade Tender Intelligence Platform for Railway and Government tenders.

## Engineering Rules

1. Follow Clean Architecture.
2. Follow Domain Driven Design.
3. Use FastAPI backend.
4. Use PostgreSQL database.
5. Use Next.js frontend.
6. Use async APIs.
7. Use repository pattern.
8. Use dependency injection.
9. Generate tests with every feature.
10. Never generate mock business logic.

## AI Rules

1. Never hallucinate tender fields.
2. Return UNKNOWN when data is missing.
3. Every extraction must include confidence score.
4. Every AI output must be validated.

## Coding Rules

1. Use type hints.
2. Use Pydantic.
3. Use SQLAlchemy 2.
4. Use Alembic migrations.
5. Use structured logging.
6. No hardcoded secrets.
7. No duplicate code.

## Development Process

PLAN → REVIEW → IMPLEMENT → TEST → DOCUMENT
